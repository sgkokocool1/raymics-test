# 桌面 App（非网页）能怎么做？

结论先说：

> **飞书 / Lark 桌面客户端没有官方插件/扩展接口。**  
> 在「不入侵客户端」的前提下，**不能**像油猴那样挂在 App 里自动读气泡。  
> 你若坚持只用桌面 App、又不要改客户端，可靠路径只剩下面几类。

## 什么叫「入侵」

| 做法 | 是否入侵 | 说明 |
| --- | --- | --- |
| 改飞书安装包 / 注入 DLL / Frida Hook | 是 | 不稳定，易封号，合规风险高 |
| 直接读客户端本地加密数据库/缓存 | 是 | 私有格式，常加密，升级即失效 |
| 抓包解密飞书私有协议 | 是 | 绕过官方能力边界 |
| 开放平台机器人（服务端事件） | **否** | 官方 API，与用网页还是 App **无关** |
| 油猴挂网页版 | **否** | 只作用于浏览器，不动桌面 App |
| 系统无障碍/OCR 读屏幕上已显示的字 | 灰区 | 不改飞书文件，但脆弱、易误读，且可能违反使用条款 |

本项目**不做**前三种。

## 方案对比（你只用桌面 App 时）

### 1. 本机跑开放平台机器人（推荐，且真正可用）

你日常继续用**桌面 App** 聊天；另开一个本机小服务收飞书官方事件。

- 群里有人发消息 → 飞书服务器推给**你的本机机器人** → 存原文  
- 有人撤回 → 再推撤回事件 → 按 ID 标记  
- 你在浏览器打开 `http://localhost:3000` 看撤回列表（这只是查看面板，不是用网页版聊天）

要点：

- **不要求**你用网页版聊天  
- **不修改**桌面 App  
- 机器人必须在目标群里；群全量消息通常要敏感权限 `im:message.group_msg`（企业管理员审批）  
- 这是官方支持路径，也是桌面场景下唯一稳定的「不入侵」自动方案  

详见上级 [README.md](../README.md) 方案 B。

```bash
cd feishu-recall-viewer
cp .env.example .env   # 填 App ID / Secret / Token
npm install && npm start
# 桌面 App 照常使用；撤回内容在本机面板查看
```

### 2. 油猴 + 网页版（不适合「不能网页打开」）

只对浏览器生效。你明确说不能网页打开，则这条不适用。  
脚本见 [`feishu-recall-local.user.js`](./feishu-recall-local.user.js)。

### 3. 纯手动、零技术（一定不入侵）

- 重要内容先**转发到自己的私聊/云文档/本地笔记**  
- 或截图保存  

无自动能力，但零风险。

### 4. 系统无障碍 / 屏幕 OCR（不推荐当正式方案）

用 macOS Accessibility、Windows UI Automation，或定时截屏 OCR，去「读屏幕上已经显示的字」。

- 优点： theoretically 不改飞书程序文件  
- 缺点：窗口一改版就挂、多屏/缩放易错、图片/表情难还原、耗电、隐私面大，且可能踩使用条款  
- 本仓库**暂不提供**此类实现

## 常见误解

**「我只用桌面 App，所以机器人方案没用？」**  
错。机器人走的是**服务端事件**，跟对方/你用 App 还是网页无关。你照样用桌面 App；本机只是多跑一个归档服务。

**「有没有官方桌面插件市场？」**  
飞书开放能力主要是机器人、网页应用、小组件等，**没有**「给桌面 IM 客户端装油猴」这种模型。

**「别人做的看撤回工具是怎么做的？」**  
市面很多是：读本地库 / Hook 客户端 / 抓包，属于入侵或灰产，升级易失效，企业环境风险大。本项目明确不做。

## 建议你怎么选

| 你的约束 | 建议 |
| --- | --- |
| 必须桌面 App 聊天 + 不入侵 + 想自动 | **本机跑开放平台机器人**（方案 1） |
| 不能申请任何企业应用权限 | 只能 **手动转发/笔记**；无法稳定自动看撤回 |
| 可以偶尔开网页 | 可用油猴本地方案 |
| 接受改客户端/读本地库 | 本项目不提供 |

如果你愿意走方案 1，我可以再帮你补一份「只在本机、用 ngrok/frp 暴露 webhook」的逐步配置清单（仍然不碰桌面 App 本身）。
