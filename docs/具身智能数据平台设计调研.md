## 1. 业界优秀平台盘点

具身智能数据平台在 2025—2026 年快速成熟，逐渐形成"**采集硬件 + 云原生存储 + 数据治理引擎 + 训练回流**"的一站式格局。下表整理了代表性平台及其可借鉴的设计点。

| 平台 / 方案 | 定位 | 关键可借鉴设计 |
| --- | --- | --- |
| **智元 · 觅蜂 MEgo Engine** | 无本体采集硬件 + 数据治理引擎 | 无本体采集（手戴夹爪/头戴设备）、亚毫秒级时间同步、6D 轨迹重建、智能质量评分、真机遥操/无本体/仿真三范式覆盖 |
| **艾欧智能 EmbodiFlow** | 企业级采集+标注+训练数据平台 | 多机器人/多传感器统一接入、自动+人工质检、多格式导出（LeRobot/HDF5/MCAP）、采集硬件原生集成 |
| **腾讯云 Data Platform + 艾欧** | 云原生存储治理底座 | COS 对象存储底座 + GooseFS 缓存 + MetaInsight 智能检索（自然语言检索"抓取失败案例"）、数据血缘 |
| **阿里云 OSS + CPFS + DLF** | 存算分离数据湖 | OSS 统一底座、Vector Bucket 向量检索、CPFS 高性能热数据层、Paimon/Iceberg 湖表、Shard Update 原地更新 |
| **Hugging Face LeRobot v3 + Visualizer** | 开源数据集标准与可视化 | Parquet（表格）+ MP4（视觉）+ JSON（元数据）解耦、episode 分块、流式加载、可视化标注（VQA/bbox/keypoint）|
| **Nomadic / FPV STERA** | AI 预标注 + VLA-ready 数据 | 子任务级时间戳分割、物体识别、3D 空间跟踪、跨数据集标注标准化以支持多数据集联合训练 |
| **rFabric** | 生产级机器人数据基础设施 | dataset/model/artifact 作为**可版本化的规范对象**、生命周期状态机、统一控制面（UI/CLI/SDK 同一资源模型）、血缘与策略 |

**共性结论：**

- **存储底座统一化**：几乎所有平台都以对象存储（OSS/COS/S3/MinIO）作为 PB 级多模态非结构化数据的统一底座，配合数据湖表（Iceberg/Paimon）做结构化管理。
- **存算分离 + 热数据缓存**：训练时通过 GooseFS/Alluxio/CPFS 把热数据加速到计算侧。
- **数据即资产**：数据从"文件"升级为带元数据、质量分、血缘、版本的**规范化资产对象**。
- **AI 预标注 + 人工质检**：自动预标注将效率提升 10 倍以上，人工聚焦质检。
- **格式标准化**：统一收敛到 LeRobot / HDF5 / MCAP 等可互操作格式，解决数据孤岛。
- **闭环回流**：训练反馈指导下一轮采集策略（数据飞轮）。

## 2. 核心设计理念

1. **全链路闭环（Data Flywheel）**：采集 → 治理 → 训练 → 反馈 → 再采集，形成正向数据飞轮。
2. **数据资产化**：每条数据具备唯一 ID、结构化元数据、质量分、生命周期状态、标签、血缘、版本。
3. **存算分离**：存储用对象存储/数据湖，计算按需弹性扩展，训练前热加载。
4. **元数据驱动**：所有展示、筛选、检索、统计都基于元数据库，而非直接扫原始文件。
5. **多模态统一**：RGB/深度、点云、IMU、力触觉、关节位姿、音频、语言指令统一建模、统一时间对齐。
6. **状态机治理**：数据在明确的生命周期状态之间流转，每次流转可审计、可追溯。
7. **格式标准化 + 可互操作**：内部统一 schema，导出兼容主流开源格式。

## 4. 数据全生命周期流转

下面按照用户描述的完整链路逐环节展开设计。每条数据（以 **episode/轨迹** 为基本单元）在生命周期中经历如下**状态机**：

```
uploaded(已上传) → scanned(已入库) → raw(原始待处理) → cleaning(清洗中)
   → preprocessed(已预处理) → annotating(标注中) → annotated(已标注)
   → asset(数据资产) → in_dataset(已进数据集) → training(训练中) → archived(归档)
                     ↘ rejected(废弃) / quarantined(隔离)
```

### 4.1 原始数据采集（Collection）

- **采集来源**：真机遥操作、无本体采集（手戴夹爪 / 头戴多相机 rig）、仿真合成、第三方开源数据集导入。
- **采集内容（多模态）**：多路 RGB/深度视频、点云、6D 位姿轨迹（XYZ + Roll/Pitch/Yaw）、关节角/夹爪状态、IMU、力/触觉、音频、语言指令/任务描述。
- **关键要求**：
  - **亚毫秒级时间同步**：多设备、多传感器统一时钟源，记录每帧硬件时间戳。
  - **采集端预处理**：格式标准化、初步质检、断点续采（start/stop 触发）。
  - **元信息随行**：采集时即写入采集人、设备型号、场景、机器人形态、任务、采集批次等。
- **落地格式**：推荐采集端输出 MCAP / rosbag / HDF5，统一后转 LeRobot v3。

### 4.2 上传到存储（Upload & Storage）

- **上传网关**：支持分片上传、断点续传、多线程、边采边传；大文件走对象存储直传（预签名 URL），避免服务端中转。
- **落盘约定**：以 `raw/{org}/{project}/{batch}/{episode_id}/...` 的路径规范落到对象存储原始区（Raw Zone）。
- **完整性校验**：上传后校验 MD5/CRC、文件数量、时长、传感器路数是否齐全，生成 upload manifest。
- **冷热分层**：原始区默认标准存储，长期未访问自动降冷（生命周期策略）。

### 4.3 扫描入库（Scan & Ingest）

- **触发方式**：对象存储事件通知（如 S3 Event / OSS 事件）或定时扫描任务。
- **入库动作**：
  1. 解析文件目录结构与 manifest，识别一条 episode 的全部资产文件。
  2. **提取元数据**：时长、帧数、FPS、相机数量、分辨率、传感器种类、机器人形态、采集时间、大小、任务描述。
  3. **计算派生指标**：轨迹长度、动作维度、初步质量分（如时间戳跳变率、丢帧率、静止帧占比）。
  4. 写入**元数据库**（关系库 + 搜索引擎），状态置为 `scanned/raw`。
  5. 建立**血缘起点**（来源批次 → episode）。
- **幂等**：以文件指纹 + episode_id 保证重复扫描不重复入库。

### 4.4 数据库中的原始数据展示与筛选（Browse & Filter）

- **列表/网格视图**：分页、缩略图/首帧预览、关键字段列（时长、场景、机器人、任务、质量分、状态）。
- **详情页**：同步视频多路播放 + 传感器曲线图（关节/动作/IMU）+ 元数据面板（参考 LeRobot Visualizer）。
- **多维筛选**：按项目、批次、采集人、设备、机器人形态、场景、任务、时间范围、时长区间、质量分区间、状态、标签、模态齐全度等组合过滤。
- **性能**：筛选走 Elasticsearch 倒排 + 聚合，原始大文件懒加载（Parquet/MP4 分块流式加载）。

### 4.5 清洗（Cleaning）

- **自动清洗规则**：
  - 剔除丢帧严重、时间戳错乱、传感器缺失、时长过短/过长的片段。
  - 静止/无效片段裁剪；异常值（力/位姿突变）检测。
  - 去重（近重复轨迹检测）。
- **质量评分**：为每条数据打质量等级（A/B/C 或 0-100 分），低分自动隔离（quarantined）。
- **人工复核**：可视化标记问题片段，支持接受/拒绝/修剪。
- 清洗产物写入 **Clean Zone**，保留与原始数据的血缘。

### 4.6 预处理（Preprocessing）

- **时空对齐**：多源传感器按主时钟重采样对齐（亚毫秒级）。
- **6D 轨迹重建**：从视频/传感信号重建完整六自由度运动轨迹。
- **标准化转换**：分辨率/帧率归一、坐标系统一、动作/状态归一化统计（mean/std/min/max）。
- **格式转换**：统一转为内部标准 schema（对齐 LeRobot v3：Parquet 表格 + MP4 视频 + JSON 元数据）。
- 产物写入 **Processed Zone**，可被标注和训练直接消费。

### 4.7 标注（Annotation）

- **标注类型**：
  - 时序分割：子任务/原子动作的起止时间戳（sub-task segmentation）。
  - 语言标注：分层任务指令（task → subtask → atomic action）、VQA。
  - 空间标注：bbox、keypoint、3D 物体位置、抓取点。
  - 成败标注：success/failure、阶段（phase）标签。
- **AI 预标注 + 人工质检**：预标注模型自动生成候选，人工在标注工作台修正（bbox/keypoint 叠加在视频上拖拽），效率提升 10 倍以上。
- **标注工作流**：任务分配 → 标注 → 一审 → 二审/质检 → 通过/驳回，全程可审计。
- **一致性**：跨数据集标注标准化（统一 label schema），使多数据集可联合训练。

### 4.8 数据资产沉淀（Data Asset）

- 通过清洗、预处理、标注、质检的数据，晋升为**可用数据资产**：
  - 具备完整元数据、质量分、标注、血缘、版本、生命周期状态 = `asset`。
  - 资产不可变（immutable），修改产生新版本（版本化 + 快照）。
  - 资产可被检索、复用、组合成数据集。
- 参考 rFabric：把数据资产建模为**规范化、可版本化的对象**，UI/CLI/SDK 共用同一资源模型与状态机。

### 4.9 打标签（Tagging）

- 在资产之上叠加**多维标签**（详见[第 7 章](#7-标签体系与快速检索)）：
  - 场景标签（家庭/工厂/商超）、任务标签（抓取/折叠/开门）、物体标签、机器人形态标签、难度、光照、成败、数据来源等。
- 标签来源：人工打标、规则自动打标、模型自动打标（图像/视频理解）、继承批次标签。
- 标签与向量索引结合，支撑语义 + 结构化混合检索。

### 4.10 加入数据集并训练（Dataset & Training）

- **数据集构建**：
  - 通过筛选条件（标签 + 元数据 + 质量分）圈选资产 → 组成数据集。
  - 支持数据集版本化、train/val/test 切分、导出格式选择（LeRobot v3 / HDF5 / MCAP / Parquet manifest）。
  - `finalize()` 语义：冻结数据集、生成统计、写清单、锁定版本。
- **训练对接**：
  - 训练前将数据集热加载到缓存层（Alluxio/GooseFS/CPFS）加速。
  - 流式训练：`StreamingLeRobotDataset` 直接从对象存储流式读取，无需全量下载。
  - 记录训练任务 → 数据集 → 资产的完整血缘，训练结果（成功率/指标）回写，指导下一轮采集（飞轮闭环）。

## 5. 存储架构设计

采用**对象存储底座 + 数据湖表 + 热缓存 + 向量库**的存算分离架构。

### 5.1 分区（Zone）划分

| 区域 | 内容 | 存储类型 |
| --- | --- | --- |
| **Raw Zone** | 原始采集文件（视频/点云/bag/hdf5） | 对象存储标准/低频，长期降冷 |
| **Clean Zone** | 清洗后数据 | 对象存储标准 |
| **Processed Zone** | 预处理/标准化后（Parquet+MP4） | 对象存储 + 数据湖表 |
| **Asset/Dataset Zone** | 数据资产与数据集快照 | 数据湖表（Iceberg/Paimon），版本化 |
| **Hot Cache** | 训练热数据 | Alluxio/GooseFS/CPFS |

### 5.2 关键能力

- **冷热分层**：生命周期策略自动降冷归档，PB 级成本可控。
- **湖表管理**：Iceberg/Paimon 提供 ACID、版本、快照、时间旅行、Shard Update（原地更新指定列，避免冗余）。
- **向量存储**：把多模态数据编码为向量（如 OSS Vector Bucket / Milvus），支持语义检索、相似轨迹检索、few-shot 经验复用。
- **高性能读取**：Parquet 列存 + 投影下推 + Blob 列懒加载，训练时按需读取。

## 6. 数据模型与元数据设计

元数据库是整个平台的"大脑"，展示/筛选/检索/统计全部基于它。核心实体（关系模型）：

```
Organization ──< Project ──< Batch(采集批次) ──< Episode(轨迹, 核心实体)
                                                    │
Episode ──< SensorStream(传感器流)                   │
Episode ──< Annotation(标注)                         │
Episode ──< QualityReport(质量报告)                  │
Episode ──< AssetVersion(资产版本)                   │
Episode >──< Tag (多对多，经 EpisodeTag)             │
Dataset >──< AssetVersion (多对多，经 DatasetItem)   │
TrainingJob ──> Dataset                              │
Lineage(血缘): source → target 有向图                 │
```

**Episode 核心字段示例：**

```jsonc
{
  "episode_id": "ep_20260706_00123",
  "project_id": "proj_grasp",
  "batch_id": "batch_0416",
  "source_type": "teleop | bodyless | sim | imported",
  "robot_embodiment": "arm_6dof_gripper",
  "scene": "kitchen",
  "task": "pick up the red cup",
  "status": "asset",                 // 生命周期状态
  "duration_sec": 42.3,
  "fps": 30,
  "frame_count": 1269,
  "cameras": ["head", "wrist_left", "wrist_right"],
  "modalities": ["rgb", "depth", "imu", "force", "pose", "audio"],
  "action_dim": 7,
  "quality_score": 87,
  "quality_grade": "A",
  "collected_by": "user_42",
  "device": "MEgo-View",
  "collected_at": "2026-04-16T10:22:00Z",
  "storage": { "raw_uri": "s3://.../raw/...", "processed_uri": "s3://.../processed/..." },
  "tags": ["kitchen", "grasp", "cup", "success", "daylight"],
  "version": 3,
  "lineage_parent": "ep_20260706_00123@v2"
}
```

**多层索引策略：**
- 关系库（PostgreSQL/TiDB）：事务、强一致、精确筛选、统计聚合。
- 搜索引擎（Elasticsearch/OpenSearch）：全文/多维组合筛选、聚合看板数据。
- 向量库（Milvus/Vector Bucket）：语义与相似检索。
- 图库（NebulaGraph）：血缘关系存储与可视化。

## 7. 标签体系与快速检索

### 7.1 标签体系（Taxonomy）

采用**多维度分类标签 + 自由标签**结合：

| 维度 | 示例 |
| --- | --- |
| 场景 | 家庭 / 工厂 / 商超 / 户外 |
| 任务 | 抓取 / 放置 / 折叠 / 开门 / 倒水 |
| 物体 | 杯子 / 衣物 / 螺丝刀 / 抽屉 |
| 机器人形态 | 单臂 / 双臂 / 人形 / 移动底盘 |
| 采集范式 | 真机遥操 / 无本体 / 仿真 |
| 质量 | A/B/C 级、成功/失败 |
| 环境 | 光照、背景复杂度、遮挡 |
| 业务 | 项目、批次、客户、合规等级 |

- 标签支持**层级化**（如 `任务/操作/抓取/精抓`）。
- 标签有类型：枚举标签、布尔标签、数值区间标签、自由文本标签。

### 7.2 快速检索设计（标签如何秒级检索出数据）

1. **倒排索引**：标签 → episode 列表，写入 Elasticsearch，多标签 AND/OR/NOT 组合毫秒级返回。
2. **标签物化**：EpisodeTag 关系表 + ES doc 里冗余 tags 数组，避免多表 join。
3. **混合检索**：结构化过滤（标签/元数据/质量分） + 语义检索（向量相似） + 全文检索（任务描述、自然语言"抓取失败案例"）三路融合，重排序输出。
4. **聚合与分面（Facet）**：检索结果实时返回各标签维度的计数分面，支持逐步收敛（drill-down）。
5. **保存查询 / 智能数据集**：把一组检索条件保存为"动态数据集"，新数据满足条件自动纳入。
6. **相似检索**：给定一条 episode，用其向量找出最相似的 N 条，用于经验复用与数据补齐。

**检索接口示例：**

```
GET /api/search?tags=kitchen,grasp,success
  &quality_gte=80&status=asset&robot=arm_6dof_gripper
  &q=pick up cup&semantic=true&page=1&size=50
→ 返回：命中列表 + 各维度 facet 计数 + 总数
```

## 8. 首页总览看板设计

看板数据由**统计聚合服务**基于元数据库/搜索引擎预聚合（定时 + 增量），前端图表实时查询。围绕用户诉求设计以下核心卡片与图表：

### 8.1 核心指标卡（KPI Cards）
- 原始数据总量（条数 / 总时长小时数 / 总存储量 TB）
- 各生命周期状态数量（原始 / 清洗中 / 已标注 / 资产 / 已入数据集）
- 数据资产化率 = 资产数 / 原始数
- 累计标注时长、当日新增采集量、当日入库量

### 8.2 原始数据的分布（Distribution）
- **时间分布**：按天/周/月的采集量趋势折线图/柱状图。
- **来源分布**：真机遥操 / 无本体 / 仿真 / 导入的占比。
- **采集人 / 设备 / 批次分布**：Top N 排行榜。
- **时长分布直方图**：episode 时长区间分布。
- **地域/场景分布**：地图或热力图。

### 8.3 原始数据的分类占比（Category Ratio）
- 按**场景**分类占比（饼图/环形图）。
- 按**任务类型**分类占比。
- 按**机器人形态**分类占比。
- 按**模态齐全度**分类（多模态/部分模态）。

### 8.4 根据 Tag 的分类占比（Tag Ratio）
- 各标签维度的数量占比（堆叠柱状 / 树图 Treemap）。
- 标签共现分析（哪些标签经常一起出现，桑基图/关系图）。
- 标签覆盖率（已打标 vs 未打标）。
- 成功/失败标签占比（质量视角）。

### 8.5 原始数据的执行情况（Execution / Pipeline Status）
- **处理流水线漏斗图**：上传 → 入库 → 清洗 → 预处理 → 标注 → 资产 → 数据集 各环节转化率与流失。
- 各环节**处理进度/积压**（清洗队列、标注队列长度）。
- **任务执行状态**：扫描任务、清洗任务、标注任务、训练任务的成功/失败/进行中统计。
- 采集**成功率 / 失败率**（success vs failure 标签）随时间变化。
- 质量分分布（A/B/C 级占比）。

### 8.6 数据集与训练视角
- 数据集数量、各数据集规模、版本情况。
- 训练任务与数据集的血缘、训练指标回流（成功率、模型表现），支撑数据飞轮决策。

> 建议看板支持时间范围、项目、场景等全局筛选器联动；所有图表可点击下钻到对应的数据列表（图表 → 筛选列表 → 详情，形成统一导航）。

## 10. 分阶段实施路线

> 按能力构建的**依赖顺序**（非时间估算）划分，每个阶段建立在前一阶段之上。

**阶段一：存储与入库底座**
- 对象存储 + 分区规范（Raw/Clean/Processed/Asset）。
- 上传网关（分片/断点/直传）+ 完整性校验。
- 扫描入库服务 + 元数据库（PostgreSQL + Elasticsearch）。
- 基础数据浏览、筛选、详情（多路视频 + 曲线同步播放）。

**阶段二：治理流水线**
- 清洗规则引擎 + 质量评分 + 隔离机制。
- 预处理（时空对齐、6D 轨迹重建、标准化、LeRobot 转换）。
- 工作流编排（Airflow/Argo）串联流水线。
- 血缘记录（图库）。

**阶段三：标注与资产化**
- 标注工作台（时序分割 / bbox / keypoint / 语言 / 成败）。
- AI 预标注模型 + 人工质检工作流。
- 资产版本化与生命周期状态机。

**阶段四：标签、检索与数据集**
- 标签体系 + 自动/人工打标。
- 混合检索（结构化 + 全文 + 向量）+ 分面 + 保存查询。
- 数据集构建、版本化、切分、多格式导出。

**阶段五：训练回流与看板**
- 训练任务编排 + 热缓存加速 + 流式读取。
- 训练指标回流 + 数据飞轮。
- 总览看板（分布 / 分类占比 / 标签占比 / 执行漏斗 / 训练视角）。
